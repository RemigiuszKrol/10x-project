Przeanalizuj plik {{file}} i przedstaw bardzo krótką analizę jego kluczowych aspektów i funkcjonalności

Przeanalizuj następujące aspekty:
1. **Typ i przeznaczenie pliku** - Jaki to typ pliku (komponent, serwis, hook, utility, itp.) i jakie jest jego główne przeznaczenie?

2. **Główne funkcjonalności** - Wymień bardzo krótko (1-2 zdania na funkcję) główne funkcje, komponenty, klasy lub eksporty:
  - Dla komponentów: jakie props przyjmuje, co renderuje, jakie ma stany
  - Dla serwisów: jakie metody/funkcje eksportuje, jakie operacje wykonują
  - Dla hooks: jakie wartości zwraca, jakie efekty uboczne ma\
  - Dla utilities: jakie funkcje pomocnicze zawiera
  
3. **Zależności** - Jakie główne zależności (importy) wykorzystuje?

4. **Kluczowe wzorce** - Jakie wzorce projektowe lub architektoniczne są zastosowane?

5. **Interakcje z innymi częściami systemu** - Z jakimi innymi modułami/plikami współpracuje?\n\nFormat odpowiedzi powinien być zwięzły i strukturalny:
```
## File Analysis: [nazwa pliku]
**Type:** [typ pliku]
**Purpose:** [krótki opis przeznaczenia]

**Key Functionalities:**
- [Funkcja 1] - [krótki opis]
- [Funkcja 2] - [krótki opis]
- [Funkcja 3] - [krótki opis]

**Main Dependencies:**
- [Zależność 1]
- [Zależność 2]

**Design Patterns:**
- [Wzorzec 1]

**Integration Points:**
- [Punkt integracji 1]
```

Pamiętaj:
- Bądź zwięzły - każdy punkt powinien być maksymalnie 1-2 zdania
- Skup się na kluczowych aspektach, nie na szczegółach implementacji
- Jeśli plik jest bardzo prosty, odpowiednio skróć analizę
- Jeśli plik jest bardzo złożony, skup się na najważniejszych elementach
